Circles

Hack Western II Project

Platform:

Android


Built With:

Java, XML, Android Studio


Start to End Date:

November 2015 - Present


Project Goal:

To develop a infinite, skill-demanding android game that has an original concept while also learning XML and developing on the Android platform.


Description:

Circles is an Android game that was conceived at Hack Western II. The game is a move and dodge game and continues until the player fails to dodge 'circles' of other colours (i.e. the player controlled 'circle' collides with another 'circle' of a different colour). Players move by tilting the Android device and gain points when 'circles' of the same color collide. 'circles' of random colours are ejected from the player controlled 'circle' when the player taps the screen. This project was developed using existing knowledge in Java, OOP principals and working knowledge in XML. My main roles were implementing the object 'circles', collisions system, ejection of 'circles', their motion and graphical display. Gained experience in the Android, in particular graphics library and Android Studio IDE.


Current Work:

Although a working version was submitted to Hack Western, my partner and I are still working to improve game experience and add new game elements, with the goal of publishing the project on Google Play. Currently working on increasing game difficulty.

Check out the project on: Github and the submitted version on: Devpost

Orbit Game

SE 101 Project

Platform:

Arduino, Tiva C Launchpad, Orbit Booster Pack


Built With:

C, Energia


Start to End Date:

September 2015 - November 2015


Project Task:

To develop a program on the the Launchpad using the Orbit Booster Pack.


Description:

The proposed project is a dodging game that utilises the accerometer of the Launchpad to move a sprite. The image is to be displayed on the small screen of the Launchpad. The goal is to dodge falling blocks as they get progressively faster and greater in number. My roles were to implement falling blocks, game difficulty, display, score system and the conditions upon when the user loses. A cool feature I worked on is the shield power up so that it can protect the sprite from one block at the press of a button. I also programmed switches of the booster pack to navigate between menu, game, scoreboard and credits page.

Check out the project on: Github

Land Battle Chess

ICS 4U7 - Internal Assessment (Individual)

Platform:

Windows


Built With:

Java, Object Oriented Programming Model


Start to End Date:

November 2014 - March 2015


Project Task:

To computerize a 2 player version of the chinese board game "Luzhanqi" (land battle chess) that is fully functional.


Description:

A description of the game can be found here. This project uses the Java Swing class to display the game board and accepts console input to select pieces on the board. An added feature of the product is the ability to undo turns up to the start of the game, to allow players to develop and rethink strategies. This feature is achieved with the use of stacks implemented with linked lists. Exception handling was also implemented for run-time errors with user input. The development procedure is documented (including planning, timeline, programming techniques, video presentation and evaluation). Documentation of code was done in Oracle standards with Javadoc.

Check out the project and its documents on: Github
